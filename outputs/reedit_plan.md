# 影片重剪規格

來源主題：使用 GitHub 檢視最近的 notebook 與管線變更，並說明變更內容

## 剪輯目標

- 移除原始聲音。
- 改用冷靜、理性、科普感的中文旁白；不模仿特定真人聲紋。
- 加上繁體中文字幕。
- 讓內容比原版更完整：補上背景、操作重點、判讀方式、風險提醒與結論。
- 影片長度可壓在 4 到 6 分鐘；若原片畫面較短，可用停格、局部放大、章節標題與重點字幕補節奏。

## 建議結構

1. 開場 0:00-0:20
   - 說明這支影片要看什麼：GitHub 近期變更、notebook 差異、pipeline 更新。
   - 畫面可放 GitHub repo 首頁或 PR/commit 清單。

2. 背景 0:20-0:55
   - 解釋 notebook 與 pipeline 在專案中的角色。
   - 若畫面有檔案樹，放大 notebook、workflow、pipeline 相關路徑。

3. GitHub 變更總覽 0:55-1:45
   - 帶觀眾看 commit、PR、changed files 或 diff。
   - 標出「哪些檔案被改」、「改動集中在哪裡」、「是否影響執行流程」。

4. Notebook 變更 1:45-2:45
   - 說明 notebook 的主要調整：資料載入、前處理、分析步驟、輸出結果或說明文字。
   - 若 notebook diff 很難讀，建議用靜態摘要字幕補充。

5. Pipeline 變更 2:45-4:05
   - 說明 workflow / CI / ETL / training pipeline 的調整。
   - 重點放在觸發條件、依賴版本、執行步驟、輸出位置、失敗時的影響。

6. 判讀與注意事項 4:05-4:50
   - 提醒觀眾檢查測試結果、資料版本、環境變數與權限設定。
   - 若 GitHub Actions 有狀態圖示，放大成功或失敗訊息。

7. 結尾 4:50-5:15
   - 總結這次變更的價值：更容易追蹤、更自動化、更可重現。
   - 收在「下次看 PR 時，可以先看 changed files，再看 pipeline 結果」。

## 視覺處理

- GitHub diff 畫面建議加 110%-140% 局部放大，避免程式碼太小。
- 關鍵路徑與行數可用半透明黃框標註。
- 章節轉場簡短即可，避免過度特效。
- 字幕使用白字、黑色半透明描邊或陰影，放在畫面下方安全區。

## 音訊處理

- 原音軌降到靜音。
- 旁白音量目標：-16 LUFS 左右。
- 背景音樂可選，若使用，音量控制在旁白以下約 20 dB。
